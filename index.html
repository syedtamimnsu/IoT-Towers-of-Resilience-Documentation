Overview

This documentation translates the conference paper into an implementation-first guide: architecture, components, data flow, deployment scenarios, and evaluation ideas.

Problem Context

Climate change increases the frequency and intensity of extreme weather, creating threats to environment, health, economy, and security—especially in vulnerable regions.

Design principle: fast sensing, fast logging, and fast actuation — without expensive infrastructure.

Research Contribution

A distributed IoT system with multiple sensing towers (NodeMCU ESP8266 + sensors/actuators) connected to a central hub (Raspberry Pi Pico) that stores telemetry in Firebase and supports environmental safety actions such as fire suppression and irrigation control.

System Architecture

A multi-tower sensing network feeds a central hub, which pushes data to cloud storage and supports alerts and control actions.

Implementation Guide

Practical steps to replicate the prototype and extend it into a deployable product.

Build Steps

  1. Assemble tower node: ESP8266 + sensors + actuators + power
  2. Calibrate sensor thresholds (moisture, fire, vibration)
  3. Implement local triggers (pump, spray, cover)
  4. Connect nodes to hub (Pico) and validate message format
  5. Push logs to Firebase and confirm live updates
  6. Design dashboard + alerts for critical events

Deployment Notes

  • Use enclosures + weather sealing
  • Consider solar + battery for remote sites
  • Add offline buffering when internet is unstable
  • Secure credentials (rotate Firebase keys)
  • Maintain an audit log for actions (pump/spray/alerts)
Tip: reduce false positives with sensor fusion (e.g., fire + temp spike).
